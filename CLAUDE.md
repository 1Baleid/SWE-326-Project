# CLAUDE.md - Project Overview

## What This Project Is

This is a **software testing coursework project** for **SWE326: Software Testing** at King Fahd University of Petroleum & Minerals. The project involves testing a real-world events and conference management web application called **Indico**.

## System Under Test (SUT)

- **Application:** Indico - Events and Conference Management System
- **URL:** https://swe326.online
- **Documentation:** https://getindico.io/ and https://indico.global/
- **Modules Being Tested:**
  - Events Module
  - Room Booking System

## Project Phases

| Phase | Description |
|-------|-------------|
| **Phase 1** | Requirements Extraction - Extract all requirements from user manuals and document in System Test Plan |
| **Phase 2** | Manual Testing - Design and execute test cases based on extracted requirements, report bugs with screenshots |
| **Phase 3** | Automated Testing - Implement automated tests using Testsigma/Selenium and apply additional testing methodologies |
| **Phase 4-5** | Additional phases (details to be provided later) |

## Team Organization

- **Each student acts as team leader for one phase** (rotating leadership)
- **All team members contribute to all phases** - work distribution is flexible within the team
- Leadership role coordinates the phase; actual work is collaborative

## Testing Tools

- **Testsigma** - Automated testing platform
- **Selenium** - Web automation framework
- **Manual Testing** - Browser-based testing with documentation

## Key Deliverables

Each phase should produce documentation including:
- History and Introduction
- Objectives
- System Requirements
- System Test Cases
- Execution Results
- Bug Reports with Screenshots

## How to Work on This Project

1. **Read the project specification:** `252-SWE326-Project-Details.pdf`
2. **Study the user manuals** in `Project Resources/System User Manual-*/`
3. **Access the live application** at https://swe326.online
4. **Use templates** provided for test plans and documentation
5. **Document everything** with screenshots as evidence

## Evaluation Criteria

- **Completeness** - All required sections covered
- **Clarity** - Logical structure and easy-to-follow presentation
- **Coverage** - Breadth of testing activities
- **Tool Usage** - Evidence of using professional testing tools

## Notes for Development

- This is a testing project, not a development project
- No source code to build - focus is on testing the live SUT
- All test artifacts should be documented in Word/PDF format
- Screenshots are required as evidence of testing activities
